# Skill：手冊章節製作（撰寫/改寫）

## 目標

用一致的「歐萊禮式節奏」產出/更新一章內容，讓讀者能快速抓到**起手式**、**trade-off** 與**上線風險**，並可被工程團隊拿去做決策與落地。

## 輸入 / 輸出

- **輸入**：一個明確要解的問題（章節主題）、目標讀者、預期交付物（schema/API/流程/指標）
- **輸出**：
  - 一個新的或更新的章節檔（`rag-handbook/<章節目錄>/<檔名>.md`）
  - 章節結尾必含：`## 本章小結`、`## 延伸閱讀`

## 依據（本 repo 的單一真相）

- **寫作規範**：`rag-handbook/STYLEGUIDE.md`
- **章節模板**：`rag-handbook/templates/chapter-template.md`
- **本書慣例/閱讀節奏**：`rag-handbook/00-導讀/00-手冊使用方式.md`、`rag-handbook/index.md`

## 章節骨架（必備）

所有章節至少包含以下區塊（順序固定）：

- `# <章名>`
- `## 你將學到（Learning Objectives）`：3–6 點、可驗收能力
- `## 本章地圖`：適合用在 / 你會做出 / 最可能踩雷
- 主文：多個 `##` / `###`，節奏是「先講結論 → 展開理由 → 給起手式 → 說例外」
- `## 本章小結`：5–9 點「可操作結論」
- `## 延伸閱讀`：3–7 項（本手冊章節＋官方文件）

（可直接從 `rag-handbook/templates/chapter-template.md` 起手。）

## 側欄（admonitions）使用

依 `rag-handbook/STYLEGUIDE.md` 的固定用法：

- **`> [!TIP]`**：起手式（可直接採用的參數/流程/預設值）
- **`> [!WARNING]`**：上線風險（越權、外洩、成本、尾延遲、不可回放）
- **`> [!CAUTION]`**：常見誤解（很容易做錯且代價很大）
- **`> [!NOTE]`**：概念定義/上下文（短、不中斷主線）

規則：

- 側欄只放「不中斷主線也值得看」的內容
- 一個側欄最多約 6 行；超過就拆或回主文

## 寫作檢查清單（提交前）

- **骨架完整**：至少含「你將學到／本章地圖／本章小結／延伸閱讀」
- **結論先行**：每個策略先給能直接採用的起手式，再談例外與調參
- **工程語言**：能對應到工件（schema、API、流程、指標、責任邊界）
- **名詞一致**：以 `rag-handbook/00-導讀/01-名詞表與縮寫.md` 為準
- **內部連結可搬移**：使用相對路徑（避免硬編完整網址）
- **引用/來源可追溯**：必要時標註文件名/章節/頁碼/URL（詳見寫作規範）

## 常見坑

- **只堆知識、不給起手式**：讀者看完仍不知道怎麼做
- **只講最佳狀況**：不上線風險、例外與降級策略
- **章節沒交付物**：最後變成概念文，無法做 code review / 驗收
